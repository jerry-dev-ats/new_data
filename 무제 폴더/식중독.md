# 식품안전나라 OpenAPI 응답값 전체 저장 및 CSV 변환 가이드

## 1. 목적

이 문서는 식품안전나라 OpenAPI의 `I2848` 서비스를 호출하여 API 응답값 전체를 수집하고, 수집한 데이터를 CSV 파일로 저장하기 위한 요청 명세와 파이썬 실행 코드를 정리한 문서입니다.

해당 API는 응답 파일 타입으로 `xml` 또는 `json`을 지원합니다. CSV를 직접 반환하는 방식은 제공되지 않으므로, 일반적으로 `json`으로 요청한 뒤 파이썬에서 CSV로 변환합니다.

---

## 2. API 기본 정보

| 구분           | 값                                                                                  |
| -------------- | ----------------------------------------------------------------------------------- |
| API 구분       | 식품안전나라 OpenAPI                                                                |
| 기본 요청 주소 | `http://openapi.foodsafetykorea.go.kr/api/keyId/serviceId/dataType/startIdx/endIdx` |
| 샘플 요청      | `http://openapi.foodsafetykorea.go.kr/api/sample/I2848/xml/1/5`                     |
| 서비스 ID      | `I2848`                                                                             |
| 권장 응답 타입 | `json`                                                                              |
| 최종 저장 형식 | `csv`                                                                               |

---

## 3. 요청 URL 형식

### 3-1. 추가 요청인자가 없는 경우

```txt
http://openapi.foodsafetykorea.go.kr/api/{인증키}/{서비스명}/{요청파일타입}/{요청시작위치}/{요청종료위치}
```

### 3-2. 추가 요청인자가 있는 경우

```txt
http://openapi.foodsafetykorea.go.kr/api/{인증키}/{서비스명}/{요청파일타입}/{요청시작위치}/{요청종료위치}/{변수명=값&변수명=값2}
```

### 3-3. 실제 요청 예시

```txt
http://openapi.foodsafetykorea.go.kr/api/sample/I2848/json/1/5
```

```txt
http://openapi.foodsafetykorea.go.kr/api/sample/I2848/json/1/5/OCCRNC_YEAR=2024&OCCRNC_MM=01&OCCRNC_AREA=서울
```

## 4. 요청 인자

| 번호 | 변수명        | 타입   | 필수 여부 | 설명           | 값 설명                       |
| ---: | ------------- | ------ | --------- | -------------- | ----------------------------- |
|    1 | `keyId`       | STRING | 필수      | 인증키         | OpenAPI에서 발급된 인증키     |
|    2 | `serviceId`   | STRING | 필수      | 서비스명       | 요청 대상 서비스명            |
|    3 | `dataType`    | STRING | 필수      | 요청 파일 타입 | `xml`, `json`                 |
|    4 | `startIdx`    | STRING | 필수      | 요청 시작 위치 | 정수 입력                     |
|    5 | `endIdx`      | STRING | 필수      | 요청 종료 위치 | 정수 입력                     |
|    6 | `OCCRNC_YEAR` | STRING | 선택      | 발생년         | `YYYY` 형식                   |
|    7 | `OCCRNC_MM`   | STRING | 선택      | 발생월         | `MM` 형식                     |
|    8 | `OCCRNC_AREA` | STRING | 선택      | 발생지역       | 예: `서울`, `부산`, `경기` 등 |

---

## 5. 출력 항목

API 응답의 `row` 데이터에서 아래 항목을 CSV 컬럼으로 저장합니다.

| 번호 | 항목          | 설명     |
| ---: | ------------- | -------- |
|    1 | `OCCRNC_YEAR` | 발생년   |
|    2 | `OCCRNC_MM`   | 발생월   |
|    3 | `OCCRNC_AREA` | 발생지역 |
|    4 | `OCCRNC_CNT`  | 발생건수 |
|    5 | `PATNT_CNT`   | 환자수   |

---

## 6. CSV 저장 정책

CSV 파일은 API에서 내려주는 모든 `row` 데이터를 저장합니다.

기본 CSV 컬럼은 다음과 같습니다.

```txt
OCCRNC_YEAR,OCCRNC_MM,OCCRNC_AREA,OCCRNC_CNT,PATNT_CNT
```

단, API에서 문서에 없는 추가 필드가 내려올 가능성도 있으므로, 아래 파이썬 코드는 응답 row에 포함된 모든 key를 자동으로 수집하여 CSV 컬럼으로 저장합니다.

즉, 현재 문서 기준으로는 5개 컬럼이 저장되지만, 추후 API 응답 필드가 추가되어도 누락 없이 CSV에 포함됩니다.

---

## 7. 전체 데이터 수집 방식

이 API는 `startIdx`, `endIdx` 범위를 이용해 데이터를 조회합니다.

예를 들어 한 번에 1,000개씩 조회할 경우 다음과 같이 반복 요청합니다.

```txt
1 ~ 1000
1001 ~ 2000
2001 ~ 3000
...
```

응답의 `total_count` 값을 확인하여 전체 데이터 개수를 파악하고, 전체 개수만큼 반복 요청한 뒤 모든 데이터를 하나의 CSV 파일로 저장합니다.

## 9. CSV 변환용 파이썬 코드

```python
import os
import math
import time
from urllib.parse import quote

import requests
import pandas as pd


BASE_URL = "http://openapi.foodsafetykorea.go.kr/api"
API_KEY = os.getenv("FOODSAFETY_API_KEY", "sample")
SERVICE_ID = "I2848"
DATA_TYPE = "json"
PAGE_SIZE = 1000
OUTPUT_CSV = "foodsafety_i2848_all.csv"


# 선택 필터
# 필요한 값만 입력하고, 사용하지 않을 필터는 None으로 둡니다.
FILTERS = {
    "OCCRNC_YEAR": None,  # 예: "2024"
    "OCCRNC_MM": None,    # 예: "01"
    "OCCRNC_AREA": None,  # 예: "서울"
}


def build_filter_path(filters: dict) -> str:
    """선택 파라미터를 OpenAPI URL 경로 형식으로 변환합니다."""
    active_filters = {
        key: value
        for key, value in filters.items()
        if value is not None and str(value).strip() != ""
    }

    if not active_filters:
        return ""

    query = "&".join(
        f"{key}={quote(str(value))}"
        for key, value in active_filters.items()
    )

    return f"/{query}"


def build_url(start_idx: int, end_idx: int, filters: dict | None = None) -> str:
    """API 요청 URL을 생성합니다."""
    filter_path = build_filter_path(filters or {})

    return (
        f"{BASE_URL}/{API_KEY}/{SERVICE_ID}/{DATA_TYPE}/"
        f"{start_idx}/{end_idx}{filter_path}"
    )


def request_api(start_idx: int, end_idx: int, filters: dict | None = None) -> dict:
    """API를 호출하고 JSON 응답을 반환합니다."""
    url = build_url(start_idx, end_idx, filters)

    response = requests.get(url, timeout=20)
    response.raise_for_status()

    return response.json()


def parse_response(data: dict) -> tuple[int, list[dict]]:
    """API 응답에서 total_count와 row 목록을 추출합니다."""
    if SERVICE_ID not in data:
        result = data.get("RESULT") or data.get("result") or {}
        code = result.get("CODE") or result.get("code")
        message = result.get("MSG") or result.get("msg")
        raise RuntimeError(f"API 응답에 {SERVICE_ID} 데이터가 없습니다. code={code}, message={message}")

    body = data[SERVICE_ID]

    result = body.get("RESULT", {})
    code = result.get("CODE")
    message = result.get("MSG")

    if code and code != "INFO-000":
        raise RuntimeError(f"API 오류 발생: code={code}, message={message}")

    total_count = int(body.get("total_count", 0))
    rows = body.get("row", []) or []

    return total_count, rows


def fetch_all_rows() -> list[dict]:
    """전체 데이터를 페이지네이션으로 수집합니다."""
    first_data = request_api(1, PAGE_SIZE, FILTERS)
    total_count, first_rows = parse_response(first_data)

    all_rows = first_rows[:]

    if total_count == 0:
        return all_rows

    total_pages = math.ceil(total_count / PAGE_SIZE)
    print(f"전체 데이터 수: {total_count}")
    print(f"전체 페이지 수: {total_pages}")
    print(f"1/{total_pages} 페이지 수집 완료")

    for page in range(2, total_pages + 1):
        start_idx = (page - 1) * PAGE_SIZE + 1
        end_idx = page * PAGE_SIZE

        data = request_api(start_idx, end_idx, FILTERS)
        _, rows = parse_response(data)
        all_rows.extend(rows)

        print(f"{page}/{total_pages} 페이지 수집 완료: 현재 {len(all_rows)}건")

        # 공공 API 서버에 과도한 요청을 보내지 않기 위한 대기 시간
        time.sleep(0.2)

    return all_rows


def save_rows_to_csv(rows: list[dict], output_csv: str) -> None:
    """수집한 row 데이터를 CSV로 저장합니다."""
    if not rows:
        print("저장할 데이터가 없습니다.")
        return

    df = pd.DataFrame(rows)

    # 문서에 명시된 컬럼을 앞쪽에 배치하고, 추가 응답 필드는 뒤쪽에 유지합니다.
    preferred_columns = [
        "OCCRNC_YEAR",
        "OCCRNC_MM",
        "OCCRNC_AREA",
        "OCCRNC_CNT",
        "PATNT_CNT",
    ]

    ordered_columns = [col for col in preferred_columns if col in df.columns]
    extra_columns = [col for col in df.columns if col not in ordered_columns]
    df = df[ordered_columns + extra_columns]

    df.to_csv(output_csv, index=False, encoding="utf-8-sig")
    print(f"CSV 저장 완료: {output_csv}")
    print(f"저장된 행 수: {len(df)}")
    print(f"저장된 컬럼: {list(df.columns)}")


if __name__ == "__main__":
    rows = fetch_all_rows()
    save_rows_to_csv(rows, OUTPUT_CSV)
```

## 12. 주의사항

1. 실제 사용 시 `sample` 키 대신 공공데이터 또는 식품안전나라 OpenAPI에서 발급받은 인증키를 사용해야 합니다.
2. CSV 변환을 위해서는 `dataType`을 `json`으로 요청하는 것을 권장합니다.
3. API 문서에는 CSV 직접 응답이 명시되어 있지 않으므로, CSV 파일은 파이썬에서 별도로 생성합니다.
4. 한 번에 너무 많은 요청을 보내지 않도록 페이지 요청 사이에 짧은 대기 시간을 두는 것이 좋습니다.
5. 한글이 깨지지 않도록 CSV 저장 시 `utf-8-sig` 인코딩을 사용합니다.
6. API 응답 구조가 변경될 수 있으므로, 코드에서는 문서에 없는 추가 컬럼도 자동으로 CSV에 포함되도록 처리합니다.
